## 구현, 완전탐색

대체적으로 문제가 특정한 알고리즘을 이용해서 답을 도출하는 것이 아니라 흔히들 말하는 '노가다'문제.  
많은 양의 코드 줄 수가 나오며, 그러므로 실수를 하지 않도록 정신을 차려야 한다.  
자주 사용하는 테크닉으로는 '이동'이 있는데, 여러 종류의 이동을 순서대로 진행할때는 dx,dy 배열을 만들어서 순회하는 것을 외워두면 편하다.  

```
int dx[4] = {1,-1,0,0};
int dy[4] = {0,0,1,-1};

for(int i=0;i<4;i++)
    arr[x+dx[i]][y+dy[i]];

```
이런식의 이동 코드를 많이 짜게 된다.
